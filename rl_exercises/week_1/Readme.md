# Week 1: Setup

The goal of this first exercise is to set up teams and learn about git and the workflow for future exercises. 
If you have not already set up this repository please make sure to do so now!

## Register an Exercise Team
Here are the steps to form a team and make sure we know you're working on the exercises.

### 1. Form teams of up to 3 students
Most exercises will require you to implement some of the techniques you learn during the course.
Git is one of the most widely used version control systems and allows you to easily collaborate with others on code from the same repository.

Exercises have to be handed in teams of up to 3 students. Make sure you and your team-mates are happy with each other, you will work together for the whole semester.
Make sure all of you have GitHub accounts and can log into them.
One person will have to own the repository for the exercises formally, decide who that will be. Also discuss if you are happy with a public repository or want a private one.

### 2. Set up your Repo 
You team's repo owner now creates a fork of this repository: https://github.com/automl-edu/RL-exercises
We recommend the other team members to get admin or at least maintainer rights as well. 
You will only see the first exercise week so far, but each week you can synch your fork to receive the next exercise (there are multiple ways to do this, but it is comparatively simple, see here: https://docs.github.com/de/pull-requests/collaborating-with-pull-requests/working-with-forks/syncing-a-fork).

Check out the resources in the repo! Most of them are optional, but will make your life easier. You can also already run an RL demo with a popular package and see the skeleton of the mini-RL package you will build over the next weeks. 
It's good to be aware of examples like plotting or tuning so you know what to do once you need them.

### 3. Inform us of your team composition
To show that you are familiar with the standard git add, commit and push steps, add a file called `rl_exercises/members.txt` to your repository.
The file should contain the names of all members (or usernames if you prefer your full names not to be visible) in the following way:

```
member 1: name1
member 2: name2
member 3: name3
```

Afterwards you can push to submit.
